XWork Roadmap

Here is an overview of the general goals of the next generation WebWork, called XWork. Suggestions and feature requests are encouraged to be directed to the WebWork mailing list, not here. Please feel free to fill this in as much as possible.

Core:

  • Completely divorce from dependence on J2EE - done
  • Provide additional Dispatchers: e.g. JMSDispatcher and MailDispatcher - Out of Scope
  • Replace the ActionFactory proxy idea with Interception support - done
  • Overhaul configuration system - done
  • Provide more view systems (xml/soap/swing/awt/tea/cf/php/flash/?) - Out of Scope
  • Break current taglib into two taglibs (core and ui) - Out of Scope - See Webwork 2.0
  • Abstract current taglib code out of taglibs to make available to other view systems - Out of Scope - See Webwork 2.0
  • Distribute core and view systems in different jars - done - Xwork is the core
  • Move to OGNL - done
  • JIRA issues/features
  • Fix/standardize logging - done I think
  • Self-reloading config system, detects changes in config files and reloads accordingly - done
