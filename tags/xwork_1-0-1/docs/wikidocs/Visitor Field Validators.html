The VisitorFieldValidator allows you to forward validation to object properties of your action using the objects own validation files. This allows you to use the ModelDriven development pattern and manage your validations for your models in one place, where they belong, next to your model classes. The VisitorFieldValidator can handle either simple Object properties, Collections of Objects, or Arrays.

Applying the VisitorFieldValidator

The VisitorFieldValidator is applied like any other field validator in a validation.xml file (see XW:Validation Framework).

<field name="bean">
    <field-validator type="visitor">
        <param name="context">anotherContext</param>
        <param name="appendPrefix">false</param>
        <message>bean: </message>
    </field-validator>
</field>

The context attribute determines the context to use for validating the Object property. If not defined, the context of the Action validation is propogated to the Object property validation. In the case of Action validation, this context is the Action alias. Here we see the context being overridden in the validator mapping, so the action alias context will not be propogated.

The appendPrefix attribute determines whether the field name of this field validator should be prepended to the field name of the visited field to determine the full field name when an error occurs. The default value is true. For example, suppose that the bean being validated with the VisitorFieldValidator in this case has a name property. If appendPrefix is true, then the field error will be stored under the field "bean.name". If appendPrefix is false, then the field error will be stored under the field "name".

The message for the VisitorFieldValidator gives a String which will be appended in front of validation messages added by the validations for the Object message.
